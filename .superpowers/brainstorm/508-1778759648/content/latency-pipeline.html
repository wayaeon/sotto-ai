Why 230ms feels instant

Human perception of "instantaneous" is under ~300ms for connected events (like typing). By streaming partial text word-by-word while you're still speaking and only doing a tiny final-chunk cleanup after you release the key, the user sees text flowing in real time — the 230ms tail is imperceptible because the visual feedback already started mid-sentence.

ON RealtimeSTT library
KoljaB's Python library — wraps faster-whisper with VAD, ring buffers, and streaming word-by-word output. Replaces building the entire pipeline from scratch. This is our sidecar engine.
Saves ~500ms total pipeline latency vs naive Whisper
ON Silero VAD
32ms audio chunk analysis. Detects speech start in milliseconds — so recording begins capturing real speech immediately, not silence. Hysteresis prevents flicker at word edges. Built into RealtimeSTT.
Eliminates false starts, reduces wasted audio at segment edges
ON Chunked / streaming inference
Instead of waiting for recording to end, Whisper transcribes overlapping 1–2s chunks while you're still speaking. Results stream to the UI word-by-word. Only the final tail chunk is processed after you stop.
Makes "wait for transcription" invisible to the user
ON Pre-roll ring buffer
~500ms of audio is always buffered before VAD fires. When speech is detected, we already have the first syllable. No clipped words at the start of recordings.
Eliminates clipped first words
ON Parallel LLM cleanup
Cleanup runs on each streaming chunk as it completes — not serially after transcription ends. By the time you release the hotkey, most cleanup is already done. Only the final chunk waits.
Removes ~500ms serial LLM delay
ON Model pre-warming
Whisper and VAD models load once at app startup and stay pinned in RAM. Zero cold-start latency per recording. On your 32GB machine this is trivial — model stays warm all day.
Eliminates 1–3s model load delay per recording
OPTIONAL WhisperPipe architecture
2026 research paper. Hybrid VAD + dynamic buffering + overlapping context windows. Achieves 89ms median end-to-end latency. More complex to implement but available as a future upgrade path.
Could push final latency to sub-100ms on good hardware
FUTURE NPU acceleration
Your Ryzen AI 7's XDNA NPU has no mature Whisper driver yet — but AMD is actively working on it. Architecture will make it a drop-in swap when drivers ship.
Could push CPU latency to near-GPU speeds

The Real Secret — Streaming Overlay

Both modes show a floating overlay where text appears word-by-word as you speak. By the time you release the hotkey or pause, 80% of the text is already on screen. The final 200ms "finish" is invisible because the user is watching text flow — not a blank screen. This is what makes Wispr feel magical, and we can match it exactly.
